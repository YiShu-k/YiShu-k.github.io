Coding部署

Coding仓库搭建

  • 注册账号
  • 创建项目
    A 代码托管项目
    B 项目名称（用户名）
    C 项目标识（用户名.coding.me）
  • 代码仓库
    A 新建代码仓库
    B 仓库名称（用户名.coding.me）
    C Git仓库
    D 启用README.MD文件初始化项目（其余全部默认）
    E 代码仓库克隆SSH到站点配置里，同github一样
    F 代码仓库设置
    G 部署公钥（同github将.ssh里的那个pub文件记事本打开全部复制进来，标题随便，勾全勾上）
    H 查询公钥是否成功
      ssh -T git@coding.net   #*网上说用ssh -T git@git.coding.net是错的*
    I 推送
      hexo clean
  hexo g
  hexo d

Coding网站生产

  • 项目设置（左下角）
    A 功能开关
    B 持续集成、持续部署打开
  • 个人设置（左上角）
    A 绑定邮箱
    B 绑定服务密码
    C 进腾讯云账号信息填好并且实名（下方可以直接进入腾讯云，必须实名才能发布静态网站）
  • 持续部署静态网站
    A 新建网站
    B 静态网站（其余的该选啥选啥，类型一定是静态网站，我选择HEXO无法构建）
    C 部署
    D 部署成功下方就有访问地址了

没有搜索结果
1.安装插件

$  npm install hexo-generator-searchdb --save
或者
$ cnpm install hexo-generator-searchdb --save

2.站点配置文件添加

#表示站内搜索
search:  
    path: search.xml
    field: post
    format: html
    limit: 10000

3.主题配置文件设置

local_search:
  enable: true  #设置为true
  trigger: auto  # auto /  manual，auto 自动搜索、manual：按回车[enter ]键手动搜索
  top_n_per_article: 1
  unescape: true

HEXO文章添加本地图片

  • 修改主页配置文件_config.yml中的post_asset_folder这个选项的值为true
  • 在hexo目录下（上图）执行命令npm install hexo-asset-image –save，目的：下载安装一个可以上传本地图片的插件。
  • 运行hexo new “new-poster-name”生成博文时，/source/_posts文件夹内除了new-poster-name.md文件还有一个同名的文件夹
  • 想要上传的图片复制到这个new-poster-name文件夹中，然后在new-poster-name.md中引入图片：![你想要输入的替代文字](new-poster-name/图片名.jpg)。将new-poster-name替换为你要创建的博文的名字。
